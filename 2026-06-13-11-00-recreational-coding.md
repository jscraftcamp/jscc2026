# Recreational coding
Coding just for fun - against the AI burnout.

## What is it?
Coding for fun and for the sake of doing it.
Do it as a hobby don't try to create "value".
Your focus should be about relaxing, having fun and learning.

## Why should you to it?
- If you feel burned out on your job for whatever reason (AI). Recreational coding will remind you of the fun you could have writing code just by yourself
- Recreate the fun of coding. Coding is fun, this is why we started our career as developers
- Keep your skills sharp. If you find yourself writing less and less code it helps to keep your skills "sharp" so that you are still able to solve problems.
- Learn something. Maybe you always wanted to know how an http server works. Just start implementing a very small (shitty) version of it.

## A word on AI
Even with a lot of us devs feeling burned out because of the current AI hype, AI is not banned in recreation programming.
You can still use AI if you have fun with it or maybe try out to use AI as a mentor / teacher on the topic you want to try out.

## How to start
- Create a backlog of things you want to try out, interesting blog posts you want to read and try out what they teach or just very obscure ideas.
- Find a time slot in your week when you could just have fun with your computer. This could be everything from starting from 30 mins up to a whole Sunday. Whatever fits your personal live

## Inspiration
Now I hear you asking : “I want to try this out, but I have no idea where to start”
- pick an advent of code task and solve it with a language you always wanted to try out.
- try to create a small arcade game, like Pacman, Tetris or snake in a tech stack you are interested in.
- search Reddit for recreational coding / programming
- search YouTube for recreation coding or just watch https://www.youtube.com/@TsodingDaily
