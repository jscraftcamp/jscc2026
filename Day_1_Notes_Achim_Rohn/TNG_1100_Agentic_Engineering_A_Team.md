# Agent Engineering / A Team

## Definition Agentic Engineering

You give it a task ant let it run.

The art of how to use AI in your day to day work.

## Chatting

Don't write tasks, just talk to the AI. Tasks as markdown written by AI.

## Agent Team (Thomas Tränkler)

An agent team like working with an actual teams.

PO to plan sprint/tasks.

Developers for coding tasks.

Architect for harder tasks.

Looking at Queues.

Bottleneck is the pull requests that pile up, merging with conflicts.

Using Claude teams. SSH into running session, even from the phone. Built a task/ticket list and spawn a team of agents working on them.

Can get stuck.

Agents have full access in Docker container running on a Mac mini.

Working in sprints.

Issues as markdown files in the repo, written by the Agent.

Advantage: Trying out something that you couldn't have done before. Now you can have a full team.

You still need the human input right now, to make the decisions. It can't fully run on its own.

Not writing code on your own anymore. If there is an issue in the code, you just tell the Agent.

Steering it in the right direction: Giving it a spec and vivid examples of what to build. You find out what it can't do. And then you give it specific instructions of the architecture.
