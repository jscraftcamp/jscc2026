## proving a11y compliance:

* **who is obliged?**
	* ask the legal department whether you are
	* example: companies who digitalize (a part of) their e2e business process such as appointment booking
* **what is needed?** host an a11y statement which means filling in a report
* **which report?** WCAG 2.1 is a good start, specific countries might require more

## a11y tools & resources

* https://github.com/dequelabs/axe-core
	* can be used with playwright etc.	
* manual testing guided by https://accessibilityinsights.io/ (browser extension by microsoft)
* BITV - professional a11y testing
	* website also has guidelines how to do the test: https://bitvtest.de/en/
* APG as a start: https://www.w3.org/WAI/ARIA/apg/
* frontend tests
	* user centric tests with testing library
	* using semantics ("get-by-role") etc. (good for screen readers)
	* check keyboard focus, activate zoom, ...

## thoughts on project migration towards an accessible solution  

* maybe even more of an organizational challenge
	* sometimes structural changes in company needed
	* "a11y champion" roles can help enforce the transition

## a11y common questions in trainings:

* do I really have to do this?
* what can we do to not have to do that?
* do I have to do landscape (screen)?!

-> shows how much of a challenge this is
